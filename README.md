This repository contains the complete, modern, and interactive portfolio website designed specifically for a **Data Analyst** profile. It features an interactive layout, standard responsiveness, smooth animations, and a custom **Portfolio Analytics Dashboard** visualizer built with Chart.js.

## 🛠️ Step-by-Step Guides

### 1. Generating the PDF Resume
Recruiters often prefer PDF downloads over Word documents.
1. Open the original `Nikita resume editable.docx` in Microsoft Word.
2. Go to **File** -> **Save As...**
3. Select **PDF (*.pdf)** from the format drop-down list.
4. Save the file directly inside this `portfolio` folder as: **`Nikita_Zade_Resume.pdf`**
5. Both DOCX and PDF download buttons will now work instantly!

---

### 2. How to Host on GitHub Pages (Free) 🚀
To send your portfolio to recruiters and add it to your LinkedIn profile, you need a live website URL. GitHub Pages is free and takes less than 5 minutes to set up:

#### Step A: Create a GitHub Account & Repository
1. Go to [github.com](https://github.com/) and log in (or register).
2. Click **New** (or "+" in the top-right corner) to create a new repository.
3. Name your repository `portfolio` (or anything you prefer).
4. Select **Public** (important: must be Public for GitHub Pages to be free).
5. Leave "Add a README file" unchecked, then click **Create repository**.

#### Step B: Upload Your Files
1. On your new repository page, click the **"uploading an existing file"** link under the quick setup section.
2. Drag and drop all files from this `portfolio` folder (make sure to select `index.html`, `style.css`, `script.js`, `Nikita_Zade_Resume.docx`, and your newly created `Nikita_Zade_Resume.pdf`).
3. Scroll to the bottom and click **Commit changes**.

#### Step C: Enable GitHub Pages
1. In your repository on GitHub, click the **Settings** tab (gear icon at the top menu).
2. In the left sidebar, under the "Code and automation" section, click **Pages**.
3. Under "Build and deployment", set the Source dropdown to **Deploy from a branch**.
4. Under "Branch", click the dropdown (currently says *None*), select **main** (or *master*), select the folder as `/ (root)`, and click **Save**.
5. Wait about 1-2 minutes. Refresh the page, and you will see a message:
   > **Your site is live at `https://<your-username>.github.io/portfolio/`**

---

### 3. Adding the Portfolio to LinkedIn 💼
Once your website is live on GitHub Pages, do the following to stand out to recruiters:

1. **LinkedIn Contact Info / Website Link**:
   - Go to your LinkedIn profile -> Click **Contact info** -> Click the edit pencil.
   - Click **Add website** -> Paste your GitHub Pages link (e.g., `https://nikitazade09.github.io/portfolio/`).
   - Choose **Portfolio** as the website type.

2. **LinkedIn Profile Introduction**:
   - Add the link directly to your "About" or "Headline" section (e.g., "Aspiring Data Analyst | Check out my live interactive dashboard portfolio: [link]").

3. **LinkedIn Featured Section**:
   - Scroll to your LinkedIn profile's **Featured** section -> Click the "+" icon -> Select **Add Link**.
   - Paste your portfolio link, add a title like *"Interactive Data Analyst Portfolio & Dashboard"*, and write a short description highlighting your key project metrics (1,000+ employee records analyzed).

4. **Sending to Recruiters**:
   - Include the link in your cold outreach messages and email signatures:
     > *"You can view my interactive portfolio dashboard and project outcome analysis live at [your-link]."*
